Docker en Kubernetes

Let's orchestrate the virtual world of containers.

Wat is het en waarom zou je het willen combineren?

kubernetes maakt orkestratie mogelijk, dit betekent een automatiseren en mannagment van containers. Eigen aan containers is dat het een autonoom stuk code bevat die zelden onveranderlijk zijn en kunnen binnen enkele seconden worden opgestart. (Manueel of autonoom)
Kubernetes start de containers op, zorgt voor load balans, failovers, routing van netwerkverkeer en upscaling/downscaling van alle resources.
De sterkte van kubernetes is dat het ingrijpt als o.a een container vastloopt, het zorgd dan tijdelijk voor een andere container zodat er weinig of geen probleem merkbaar is.
Wat kubernetes uniek maakt is dat het de menselijke factor als zwakste schakel uitschakelt. Stel dat je een paar honderd containers hebt draaien, Als je deze allemaal manueel zal beheren is een fout niet uit te sluiten.
Door deze processen volautomatisch te laten lopen kan je zulke fouten voorkomen, daarnaast zal het je ook veel tijd besparen.

Docker



Docker is software dat gemaakt is voor systeembeheerders en ontwikkelaars.
Het is een manier voor het ontwikkelen, verschepen en uitvoeren van software door het te beheren in een virtuele
Docker maakt gebruik van containerisatie-technologie die software gaat verpakken en isoleren waardoor de toepassing minder afhankelijk word van de host.
Het is dus betrouwbaarder, eenduidiger en kan eenvoudiger worden verscheept en uitgevoerd op verschillende omgevingen.
Voor ontwikkelaars is dit een eenvoudige manier om applicaties te bouwen, distribueren en schalen ongeacht de host waar de container op draait.

Hoe het project is opgezet

In dit project gaan we Docker, docker-compose en Kubernetes installeren op nodes.
Nodes zijn de besturingssystemen waarop Docker draait en die beheerd worden met Kubernetes.
Dit kan Windows, Chrome, Mac of Linux (met of zonder grafische omgeving) zijn, maar in dit project werken we uitsluitend met Linux.
Linux biedt namelijk een stabiele en flexibele omgeving die ideaal is voor containerisatie en orkestratie.
In het project zijn drie workernodes opgezet waarop Docker ge√Ønstalleerd wordt. Deze nodes draaien containers die de basis vormen van de gedistribueerde infrastructuur.
Daarnaast is er een masternode geconfigureerd, waar Kubernetes draait.
Deze masternode speelt een cruciale rol, omdat het de centrale hub is voor het beheren en co√∂rdineren van alle containers in pods op de workernodes.

De images van de Docker containers worden op twee verschillende manieren verkregen:
  • Door ze zelf te bouwen met behulp van een Dockerfile of docker-compose. Dit geeft ons volledige controle over de configuratie en functionaliteit van de containers.
  • Door ze te halen uit een publieke repository, zoals Docker Hub, die een breed scala aan vooraf geconfigureerde images biedt.
  • Uit de eigen repository, wat handig is voor het hosten en beheren van interne of aangepaste images.

Door gebruik te maken van deze methoden kunnen we een flexibele en schaalbare infrastructuur bouwen die zowel standaardoplossingen als maatwerk ondersteunt.
Dit project biedt niet alleen praktische ervaring met containerisatie, maar ook inzicht in hoe Kubernetes clusters worden ingezet om complexe applicaties te beheren.

Dit alles maakt het mogelijk om een robuuste en toekomstbestendige infrastructuur te bouwen die klaar is voor productiegebruik.
